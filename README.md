# HTR-context-OCR

This README documents how to run every script in `python_scripts/` and `shell_scripts/`.

**Conventions**
Run commands from the repo root:

```bash
cd /scratch/project_2017385/dorian/HTR-context-OCR
```

All Python examples use `python3`. No `pixi` commands are used here.

**Python Scripts**

**`python_scripts/align_two_graph_text_blocks.py`**  
Purpose: Align prediction text blocks for a small set of target graph cases using line endpoints from chrF score matrices.

Run (all flags):
```bash
python3 python_scripts/align_two_graph_text_blocks.py
```

Flags:
- No CLI flags. Configuration is via in-file constants:
- `PROJECT_ROOT` repo root path used to import evaluation utilities.
- `SCORES_PKL` path to the `scores.pkl` file generated by `python_scripts/compare.py` or `shell_scripts/run_compare.sh`.
- `TARGET_GRAPHS` list of `*_graph.png` filenames to process.
- `WINDOW_SIZE` and `WINDOW_STRIDE` must match the values used when creating `scores.pkl`.
- `OUTPUT_DIR` output folder for adjusted text and reports.

Outputs:
- `results/aligned_text_blocks_two_cases/<graph>_adjusted_pred.txt`
- `results/aligned_text_blocks_two_cases/<graph>_alignment_report.json`
- `results/aligned_text_blocks_two_cases/summary.json`

**`python_scripts/churchbook_churro_infer.py`**  
Purpose: Run Churro OCR on local churchbook images and save XML + plain-text markdown outputs.

Run (all flags, vLLM example):
```bash
python3 python_scripts/churchbook_churro_infer.py \
  --backend vllm \
  --input-dir /scratch/project_2017385/dorian/Churro_churchbooks/churchbook_images \
  --output-root /scratch/project_2017385/dorian/Churro_churchbooks/results/churchbook_results \
  --model-id stanford-oval/churro-3B \
  --system-message "Transcribe the entiretly of this historical documents to XML format." \
  --max-new-tokens 20000 \
  --temperature 0.6 \
  --device auto \
  --vllm-base-url http://localhost:8000/v1 \
  --vllm-model churro \
  --vllm-api-key "${OPENAI_API_KEY:-EMPTY}" \
  --vllm-timeout-seconds 600 \
  --max-concurrency 1 \
  --max-images 0 \
  --skip-existing
```

Flags (all CLI flags):
- `--backend` selects inference path: `transformers` or `vllm`.
- `--input-dir` directory containing image files.
- `--output-root` root directory for output markdown files.
- `--model-id` Hugging Face model id for transformers backend.
- `--system-message` system prompt added to each request.
- `--max-new-tokens` max tokens generated per image.
- `--temperature` sampling temperature, `0` disables sampling.
- `--device` device for transformers backend: `auto`, `cpu`, or `cuda`.
- `--vllm-base-url` base URL for the vLLM OpenAI-compatible endpoint.
- `--vllm-model` served model name (as exposed by vLLM).
- `--vllm-api-key` API key for vLLM endpoint (defaults to `OPENAI_API_KEY` or `EMPTY`).
- `--vllm-timeout-seconds` HTTP timeout per request.
- `--max-concurrency` in-flight request cap for vLLM backend.
- `--max-images` cap number of images, `0` means all images.
- `--skip-existing` skip images when both output files already exist.

Outputs:
- `xml_results/<image_stem>_xml.md`
- `_pure_text_results/<image_stem>_pure_text.md`

**`python_scripts/compare.py`**  
Purpose: Sliding-window chrF comparison between gold and predicted text to produce a `scores.pkl`.

Run (all flags):
```bash
python3 python_scripts/compare.py \
  --window-size 100 \
  --window-stride 50 \
  --runfile-json /scratch/project_2017385/dorian/churro_finnish_dataset/run_results/dev_split/outputs.json \
  --output /scratch/project_2017385/dorian/Churro_copy/results/custom_churro_infer_dev_run1/vllm/dev/scores.pkl
```

Flags (all CLI flags):
- `--window-size` sliding window size in characters.
- `--window-stride` stride between window positions.
- `--runfile-json` path to `outputs.json` containing reference and prediction text.
- `--output` path to the pickled `scores.pkl` output.

Outputs:
- `scores.pkl` containing `{fname, scores, ref, pred}` entries.

**`python_scripts/custom_python_script.py`**  
Purpose: Custom module entrypoint that delegates to the Churro CLI.

Run (all flags used by the benchmark path in this repo):
```bash
python3 -m custom_python_script benchmark \
  --system finetuned \
  --engine churro \
  --dataset-split test \
  --input-size 0 \
  --max-concurrency 4
```

Flags (forwarded to Churro CLI):
- `--system` selects the system preset (e.g., `finetuned`).
- `--engine` model engine key used by the Churro CLI.
- `--dataset-split` split to benchmark (e.g., `dev`, `test`).
- `--input-size` optional cap on number of samples, `0` means all.
- `--max-concurrency` concurrency for the benchmark run.

Notes:
- This file does not define flags itself. It forwards all args to `churro.cli.main`.
- Use `python3 -m custom_python_script --help` and `python3 -m custom_python_script <subcommand> --help` to see the full flag set for each subcommand.

  --dataset-id stanford-oval/churro-dataset \
  --dataset-split all \

Flags (all CLI flags):
- `--backend` `transformers` or `vllm`.
- `--dataset-id` Hugging Face dataset id (default `stanford-oval/churro-dataset`).
- `--dataset-split` split name or `all` to auto-discover splits.
- `--max-samples-per-split` cap Finnish samples per split, `0` means all.
- `--model-id` model id for transformers backend.
- `--system-message` system prompt per request.
- `--max-new-tokens` max tokens per image.

- `--vllm-api-key` API key for vLLM endpoint.

- `--max-concurrency` max in-flight requests (vLLM only).
- `--output-dir` directory for markdown outputs.
- `--metrics-output-root` directory for metrics outputs, if omitted defaults to `--output-dir`.
- `--skip-existing` skip samples whose markdown output already exists.

Outputs:
